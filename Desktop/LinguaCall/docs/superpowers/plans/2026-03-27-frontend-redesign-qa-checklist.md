# 프론트엔드 리디자인 QA 체크리스트

## 목적

`2026-03-26-frontend-redesign-prd.md` 1차 구현 이후, 실제 브라우저에서 확인해야 할 핵심 QA 항목을 정리한다.
목표는 `결제 전환율 상승`을 위한 신뢰감 개선이 화면 전반에 반영되었는지 확인하는 것이다.

## 공통 확인 항목

- 모바일과 데스크톱 모두 첫 화면에서 핵심 CTA가 보인다
- 페이지 전환 시 레이아웃이 급격히 흔들리지 않는다
- 버튼, 카드, 배지, 입력창의 톤이 페이지 간 일관된다
- 깨진 문자열이나 언어 혼합이 남아 있지 않다
- 오류, 로딩, 빈 상태가 기술 문구가 아니라 사용자 친화 문구로 보인다

## 1. 로그인 화면

경로:

- `/`

확인:

- 핵심 메시지 `짧고 가볍게 꾸준히 연습`이 첫 화면에 보인다
- 시작 CTA가 즉시 보인다
- 결제, 리포트, 세션 세부 정보가 첫 화면을 방해하지 않는다
- 모바일에서 문구와 버튼이 과밀하지 않다
- 데스크톱에서 좌우 여백과 본문 폭이 어색하지 않다

## 2. OTP 인증 화면

경로:

- `/verify`

확인:

- 전화번호 입력 단계와 인증번호 입력 단계가 명확히 구분된다
- 요청 후 상태 배너가 자연스럽고 위협적이지 않다
- 오류, 성공 메시지가 읽기 쉬운 문장이다
- 모바일에서도 입력창과 버튼이 한 화면 내에서 무리 없이 보인다

## 3. 세션 홈

경로:

- `/session`

확인:

- 사용자가 첫 진입 시 무엇을 해야 할지 바로 알 수 있다
- `지금 시작`, `예약 보기`, `최근 리포트`가 즉각적으로 구분된다
- 현재 플랜과 잔여 사용량 정보가 과하지 않게 보인다
- 세션 생성 폼이 첫 화면을 압도하지 않는다
- 모바일에서도 카드가 세로 흐름으로 자연스럽게 쌓인다

## 4. 라이브 세션

조건:

- 세션 생성 후 실시간 통화 시작

확인:

- 연결 전, 연결 중, 종료 중 상태가 명확히 구분된다
- 통화 중에는 불필요한 관리 UI보다 상태와 transcript가 우선 보인다
- 종료 버튼이 명확하고, 로딩 중 상태 변화가 자연스럽다
- transcript가 없을 때의 빈 상태 문구가 어색하지 않다
- 모바일에서 종료 버튼과 상태 문구가 스크롤 없이 확인 가능하다

## 5. 결제 화면

경로:

- `/billing`

확인:

- 현재 플랜과 업그레이드 선택지가 한눈에 보인다
- Basic/Pro 차이가 단순 가격표가 아니라 선택 이유로 보인다
- 추천 플랜 강조가 과하지 않으면서도 분명하다
- Toss 결제 CTA가 잘 보이고 기술적으로 보이지 않는다
- 결제 완료 후 다음 행동으로 자연스럽게 이어진다

## 6. 리포트 화면

경로:

- `/report/:reportId`

확인:

- 총점, 요약, 추천 액션이 상단에서 빠르게 읽힌다
- 상세 분석은 읽고 싶은 경우 자연스럽게 아래로 내려가며 볼 수 있다
- 교정/어휘/유창성 카드가 나열이 아니라 이해 가능한 구조로 보인다
- 빈 데이터가 있어도 화면이 깨지지 않는다
- 모바일에서 카드 간격과 폭이 적절하다

## 7. 정책 페이지

경로:

- `/privacy`
- `/terms`

확인:

- 문서 레이아웃이 일관된다
- 인코딩 깨짐이 없다
- 제목과 섹션 구조가 읽기 쉽다
- 본문이 모바일에서도 지나치게 좁거나 넓지 않다

## 8. 인증/세션 흐름 체크

확인:

- 비로그인 사용자는 보호 경로 접근 시 `/`로 이동한다
- OTP 인증 후 `/session`으로 자연스럽게 진입한다
- 새로고침 후 세션이 유지된다
- 로그아웃 후 보호 경로 접근 시 다시 로그인 화면으로 돌아간다

## 9. 결제 전환 관점 최종 질문

아래 질문에 `예`라고 답할 수 있어야 한다.

- 처음 보는 사용자가 10초 안에 서비스가 무엇인지 이해할 수 있는가
- 결제 전에 서비스가 허술해 보이지 않는가
- 무료에서 유료로 넘어가야 하는 이유가 자연스럽게 보이는가
- 모바일에서도 결제 버튼까지의 흐름이 복잡하지 않은가
- 세션 홈과 결제 화면이 하나의 제품처럼 일관되게 느껴지는가

## 완료 기준

아래를 만족하면 프론트 리디자인 1차 QA 통과로 본다.

- `pnpm --filter lingua-call-web typecheck` 통과
- 주요 경로 수동 QA 완료
- 결제 전환을 방해하는 신뢰감 문제 없음
- 모바일과 데스크톱 모두 치명적 레이아웃 문제 없음
- 깨진 텍스트와 남은 placeholder 문구 정리 완료
