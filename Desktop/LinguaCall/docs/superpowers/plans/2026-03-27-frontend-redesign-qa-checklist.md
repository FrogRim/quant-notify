# Frontend Redesign QA Checklist

## 목적

이 문서는 `2026-03-26-frontend-redesign-prd.md` 1차 구현 이후, 실제 브라우저에서 확인해야 할 핵심 QA 항목을 정리한다. 목표는 `결제 전환율 상승`을 위한 신뢰감 개선이 화면 전반에서 일관되게 반영되었는지 확인하는 것이다.

## 공통 확인 항목

- 모바일과 데스크톱 모두 첫 화면에서 핵심 CTA가 잘 보인다.
- 페이지 전환 시 레이아웃이 급격히 흔들리지 않는다.
- 버튼, 카드, 배지, 입력창의 톤이 페이지 간 일관된다.
- 깨진 문자열이나 영어/한국어 혼합이 어색하게 남아 있지 않다.
- 오류, 로딩, 빈 상태가 기술 문구가 아니라 사용자 친화 문구로 보인다.

## 1. 로그인 화면

경로:
- `/`

확인:
- 서비스 핵심 메시지 `짧고 가볍게 꾸준히 연습`이 첫 시선 안에 들어온다.
- 시작 CTA가 명확하고 시각적으로 가장 강하다.
- 결제/리포트/세션 상세 같은 정보가 로그인 화면에 섞여 보이지 않는다.
- 모바일에서 히어로 문구, 설명, CTA가 과도하게 길지 않다.
- 데스크톱에서 좌우 여백과 본문 폭이 답답하거나 비어 보이지 않는다.

## 2. OTP 인증 화면

경로:
- `/verify`

확인:
- 전화번호 입력 단계와 인증번호 입력 단계가 명확히 구분된다.
- 인증 요청 후 상태 배너가 자연스럽고 위협적이지 않다.
- 재시도, 오류, 성공 메시지가 사용자가 이해하기 쉬운 문장이다.
- 모바일에서 입력창, 인증 버튼, 코드 입력이 한 손 사용 기준으로 무리 없다.
- 이미 인증된 사용자가 진입했을 때 부자연스럽게 남지 않고 다음 화면으로 이동한다.

## 3. 세션 홈

경로:
- `/session`

확인:
- 첫 진입 시 사용자가 무엇을 해야 할지 바로 이해할 수 있다.
- `지금 시작`, `예약 보기`, `최근 리포트` 흐름이 시각적으로 구분된다.
- 현재 플랜/남은 혜택 정보가 과하지 않게 보인다.
- 세션 생성 폼이 초기 화면을 압도하지 않는다.
- 모바일에서 카드가 세로로 자연스럽게 쌓이고 CTA 폭이 어색하지 않다.

## 4. 라이브 세션

조건:
- 세션 생성 후 실시간 통화 시작

확인:
- 연결 전, 연결 중, 종료 중 상태가 명확하게 구분된다.
- 통화 중에는 불필요한 관리 UI보다 통화 상태와 transcript가 우선 보인다.
- 종료 버튼이 명확하고, 로딩 중 상태 변화가 자연스럽다.
- transcript가 없을 때의 빈 상태 문구가 어색하지 않다.
- 모바일에서 종료 버튼과 주요 상태 문구가 스크롤 없이 확인 가능하다.

## 5. 결제 화면

경로:
- `/billing`

확인:
- 현재 플랜과 업그레이드 플랜의 차이가 한눈에 보인다.
- Basic/Pro가 단순 가격표가 아니라 용도와 대상이 보이는 제안처럼 느껴진다.
- 추천 플랜 강조가 과하지 않으면서도 명확하다.
- Toss 결제 CTA가 지나치게 기술적으로 보이지 않는다.
- 결제 완료 후 다음 행동으로 자연스럽게 이어진다.

## 6. 리포트 화면

경로:
- `/report/:reportId`

확인:
- 총점, 요약, 추천 액션이 상단에서 빠르게 읽힌다.
- 세부 분석은 읽고 싶은 사람만 자연스럽게 내려가며 볼 수 있다.
- 교정/어휘/유창성 정보가 나열형이 아니라 이해 가능한 구조로 보인다.
- 빈 데이터 또는 일부 누락이 있어도 화면이 깨지지 않는다.
- 모바일에서 카드 간 간격과 텍스트 밀도가 적절하다.

## 7. 정책 페이지

경로:
- `/privacy`
- `/terms`

확인:
- 문서형 레이아웃이 일관된다.
- 인코딩 깨짐이 없다.
- 상단 제목, 소개, 마지막 업데이트 표시가 자연스럽다.
- 본문 가독성이 충분하고 모바일에서 폭이 지나치게 넓지 않다.

## 8. 인증/세션 회귀 체크

확인:
- 비로그인 사용자는 보호 경로 진입 시 `/`로 이동한다.
- OTP 인증 완료 후 `/session`으로 자연스럽게 진입한다.
- 새로고침 후에도 세션이 유지된다.
- 로그아웃 후 보호 경로 접근 시 다시 로그인 화면으로 이동한다.

## 9. 결제 전환 관점 최종 체크

아래 질문에 `예`라고 답할 수 있어야 한다.

- 처음 보는 사용자가 10초 안에 이 서비스가 무엇인지 이해할 수 있는가?
- 결제 전, 서비스가 허술하지 않고 실제로 동작할 것처럼 보이는가?
- 무료에서 유료로 넘어가야 하는 이유가 자연스럽게 보이는가?
- 모바일에서 결제 버튼까지 가는 흐름이 복잡하지 않은가?
- 세션 홈과 결제 페이지가 하나의 제품처럼 일관되게 느껴지는가?

## 완료 기준

아래 조건을 만족하면 프론트 리디자인 1차 구현은 커밋 가능한 상태로 본다.

- `pnpm --filter lingua-call-web typecheck` 통과
- 주요 경로 수동 QA 완료
- 결제 플로우 시각적 신뢰감 문제 없음
- 모바일/데스크톱에서 치명적 레이아웃 깨짐 없음
- 깨진 텍스트와 명백한 영어 하드코딩 표면 정리 완료
