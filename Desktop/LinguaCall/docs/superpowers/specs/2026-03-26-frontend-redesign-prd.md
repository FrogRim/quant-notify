
# LinguaCall Frontend Redesign PRD

## 1. 문서 목적

이 문서는 LinguaCall의 프론트엔드 리디자인 요구사항을 정의한다.

이번 리디자인은 단순 미관 개선이 아니라, 현재 MVP의 핵심 문제인 `UI 신뢰감 부족`을 줄이고 `결제 전환율 상승`으로 이어지도록 하는 것을 목적으로 한다.

## 2. 배경

현재 서비스는 기능 기준으로는 MVP가 성립한다.

- 전화번호 OTP 로그인 가능
- Toss 결제 가능
- 세션 생성 가능
- 실시간 음성 통화 가능
- 리포트 조회 가능

하지만 UI 관점에서는 다음 문제가 남아 있다.

- 전체 화면이 기능 위주로 구성돼 제품 신뢰감이 약하다
- 핵심 경험이 한 화면에 과도하게 몰려 있다
- 로그인부터 결제까지의 전환 퍼널이 정돈되어 보이지 않는다
- 가격과 사용 가치가 충분히 설득되지 않는다
- “짧고 가볍게 꾸준히 연습”이라는 제품 정체성이 UI에 잘 드러나지 않는다

따라서 이번 리디자인은 기능 추가보다 `제품 인상`, `전환 퍼널`, `화면 구조`, `사용자 신뢰감`을 우선적으로 개선한다.

## 3. 제품 목표

### 3.1 1차 목표

- 결제 전환율 상승

### 3.2 2차 목표

- 로그인 후 첫 세션 시작률 상승
- UI 신뢰감 향상
- 서비스 첫인상 개선
- 리포트와 예약 기능의 체감 가치 상승

### 3.3 핵심 전략

결제 전환을 직접 올리기 위해, 사용자가 처음 접하는 화면부터 결제 직전 화면까지의 경험을 `친근하고, 정돈되어 있으며, 쉽게 시작할 수 있는 서비스`로 다시 설계한다.

## 4. 제품 포지셔닝

이번 리디자인에서 LinguaCall은 아래 톤으로 포지셔닝한다.

- 브랜드 톤: `부담 없이 연습하는 친근형`
- 핵심 메시지: `짧고 가볍게 꾸준히 연습`
- 제품 성격: `쉽고 경제적인 전화영어형 AI 회화 연습 서비스`

피해야 할 방향:

- 너무 시험 플랫폼처럼 딱딱한 인상
- 너무 엔지니어링 도구처럼 보이는 화면
- 너무 고가 프리미엄 서비스처럼 무거운 인상

지향점:

- 가볍게 시작할 수 있어 보이되, 허술해 보이지는 않아야 한다
- 친근하지만 값싸 보이지 않아야 한다
- 짧은 세션의 가치를 자연스럽게 납득시켜야 한다

## 5. 타깃 사용자

### 5.1 핵심 타깃

- 시험 대비는 하고 있지만 매일 길게 공부하지 않는 사용자
- 전화영어를 하고 싶지만 비용과 시간 부담이 큰 사용자
- 회화 앱을 시도했지만 피로감 때문에 오래 못 간 사용자
- 짧은 시간에 꾸준히 연습하고 싶은 사용자

### 5.2 사용자 기대

이 사용자들은 아래를 원한다.

- 지금 바로 시작할 수 있다는 확신
- 복잡하지 않다는 느낌
- 가격이 과하지 않다는 인상
- 짧게 연습해도 축적이 된다는 신뢰
- 실제로 말하기 연습이 된다는 체감

### 5.3 사용자 불안 요소

- 너무 기술적으로 보이면 신뢰가 떨어진다
- 너무 복잡하면 바로 이탈한다
- 가격이 보여도 왜 결제해야 하는지 이해가 안 되면 전환되지 않는다
- 시험용 서비스처럼 무겁게 느껴지면 재방문성이 떨어진다

## 6. 문제 정의

현재 UI 문제를 제품 관점에서 정의하면 아래와 같다.

### 6.1 구조 문제

- `ScreenSession`에 너무 많은 역할이 몰려 있다
- 라이브 통화, 세션 생성, 세션 목록, 예약 관리, 리포트 조회가 한 화면에 섞여 있다

### 6.2 신뢰 문제

- 로그인/OTP 화면이 기능적으로는 충분하지만 제품 첫인상으로는 약하다
- Billing 화면이 가격표는 보여주지만 제품 가치 설득이 약하다
- 정책 페이지와 일부 텍스트 인코딩 문제가 서비스 신뢰에 악영향을 준다

### 6.3 전환 문제

- 로그인 후 사용자가 무엇을 먼저 해야 할지 명확하지 않다
- 무료 사용과 유료 사용의 차이가 충분히 설득되지 않는다
- 결제 페이지가 “구매 화면”보다 “관리 화면”에 가까운 느낌이다

## 7. 리디자인 원칙

이번 리디자인은 아래 원칙을 따른다.

### 7.1 친근함

- 사용자가 “부담 없이 써볼 수 있겠다”고 느껴야 한다

### 7.2 가벼움

- 한 세션을 시작하기까지의 인지 부담이 낮아야 한다

### 7.3 신뢰감

- 로그인, 결제, 리포트 모두 정돈된 제품처럼 보여야 한다

### 7.4 짧은 연습의 가치 강조

- 10분, 15분 세션이 “짧지만 충분한 연습”처럼 느껴져야 한다

### 7.5 모바일/데스크톱 동등성

- 어느 한쪽이 부차적 화면처럼 보여선 안 된다

## 9. 정보 구조 방향

현재 한 화면에 몰려 있는 기능을 화면 역할 기준으로 분리한다.

### 9.1 목표 화면 구조

- `Landing / Login`
  서비스 가치 설명 + 시작 CTA
- `Verify`
  전화번호 인증 전용 단계
- `Home / Session Hub`
  오늘 할 행동 중심의 메인 화면
- `Live Session`
  통화 중 전용 화면
- `Session History / Reports`
  지난 세션과 리포트 확인 화면
- `Billing`
  플랜 설명과 결제 전환 중심 화면

### 9.2 핵심 원칙

- “무엇을 할지”와 “무슨 기록이 있는지”를 같은 화면에 과도하게 섞지 않는다
- 통화 중에는 통화에만 집중하게 한다
- 결제는 기능 설명이 아니라 선택과 설득 중심으로 구성한다

## 10. 화면별 요구사항

### 10.1 로그인 화면

목표:

- 서비스 첫인상 개선
- “쉽고 가볍게 시작할 수 있다”는 메시지 전달
- 전화번호 인증으로 이어지는 자연스러운 CTA 제공

필수 요소:

- 서비스명
- 핵심 카피
- 짧은 보조 설명
- 시작 CTA
- 가격/신뢰/가치에 대한 최소 보조 정보

원하는 인상:

- 간결하고 친근함
- 허술하지 않음
- 부담 없이 눌러볼 수 있음

### 10.2 OTP 인증 화면

목표:

- 기술적 인증 흐름을 자연스럽고 짧은 onboarding step처럼 보이게 함

필수 요소:

- 전화번호 입력
- OTP 입력
- 진행 상태
- 에러 메시지
- 다시 시도 동선

원하는 인상:

- 빠름
- 명확함
- 불안하지 않음

### 10.3 세션 홈

목표:

- 사용자가 로그인 후 가장 먼저 무엇을 해야 하는지 바로 알 수 있게 함
- “지금 시작”을 가장 쉽게 만듦
- 예약과 기록은 보조 흐름으로 배치

필수 요소:

- 오늘의 주요 CTA
- 즉시 시작
- 예약 보기
- 최근 세션 또는 최근 리포트 요약
- 현재 플랜/잔여 사용 가능량 요약

원하는 인상:

- 앱을 열자마자 바로 행동 가능
- 복잡하지 않음
- 오늘 할 일이 보임

### 10.4 라이브 세션 화면

목표:

- 통화 경험에만 집중하게 함

필수 요소:

- 연결 상태
- 마이크/오디오 상태
- 간단한 transcript 또는 핵심 표시
- 종료 버튼

제거 대상:

- 통화 중 필요 없는 목록, 가격, 과거 기록 노출

원하는 인상:

- 단순함
- 안정감
- 방해 요소 없음

### 10.5 결제 화면

목표:

- 플랜을 이해시키고 결제를 설득함
- Free와 유료의 차이를 명확히 보여줌
- 짧은 연습의 가치가 가격과 연결되도록 함

필수 요소:

- 현재 구독 상태
- 플랜별 차이
- 추천 플랜 강조
- 업그레이드 CTA
- 결제 후 기대 효과 설명

원하는 인상:

- “비싸다”보다 “합리적이다”에 가까움
- 비교하기 쉬움
- 선택 피로가 적음

### 10.6 리포트 화면

목표:

- 사용자에게 “연습이 쌓이고 있다”는 감각 제공
- 단순 점수표가 아니라 학습 피드백 경험으로 느껴지게 함

필수 요소:

- 총점과 세부 점수
- 요약
- 추천 액션
- 교정 내용
- 유창성/어휘 지표

원하는 인상:

- 코칭 받는 느낌
- 복습 동기 유도
- 다음 세션으로 이어지는 흐름

## 11. 결제 전환 관점 핵심 요구사항

이번 리디자인은 아래 전환 요구를 만족해야 한다.

- 로그인 전에도 핵심 가치가 이해되어야 한다
- 로그인 후 첫 메인 화면에서 유료 가치가 자연스럽게 보여야 한다
- 무료/유료 차이가 한눈에 보여야 한다
- 10분, 15분 세션의 가치가 충분히 전달되어야 한다
- 결제 화면은 기술적 목록이 아니라 구매 설득 화면이어야 한다
- 현재 구독 상태, 업그레이드 이유, 다음 행동이 더 분명해야 한다

## 12. 성공 지표

### 12.1 핵심 KPI

- 로그인 후 결제 페이지 진입률 상승
- 결제 페이지에서 checkout 시작률 상승
- checkout 시작 후 성공률 유지 또는 상승
- 첫 세션 생성률 상승
- 첫 세션 시작률 상승

### 12.2 보조 지표

- 로그인 직후 이탈률 감소
- billing 페이지 체류시간의 질적 개선
- 리포트 재방문률 상승
- 예약 세션 생성률 상승

### 12.3 정성 지표

- “허술하다”는 인상 감소
- “쉽게 시작된다”는 피드백 증가
- “짧게 써도 괜찮다”는 인식 형성

## 13. 비기능 요구사항

- 모바일과 데스크톱 모두 동등하게 자연스러워야 한다
- 화면 간 톤과 컴포넌트 규칙이 일관돼야 한다
- 로딩, 에러, 성공 상태가 더 읽기 쉽게 정리되어야 한다
- 인코딩 깨짐 텍스트는 전면 정리 대상이다
- 접근성 기본 수준을 만족해야 한다

## 14. 우선순위

### 14.1 1차 구현 우선순위

1. 로그인/인증 경험 재설계
2. 세션 홈과 라이브 화면 분리
3. 결제 페이지 재설계
4. 리포트 페이지 재설계

### 14.2 2차 구현 우선순위

5. 정책/정적 페이지 정리
6. 공통 UI 시스템 정리
7. 세션 이력/예약 관리 화면 정리

## 15. 디자인 시스템 방향

디자인 톤은 아래 방향을 따른다.

- 친근함
- 명료함
- 따뜻한 신뢰감
- 가벼운 사용감

피해야 할 스타일:

- 지나치게 기업형 B2B 도구처럼 보이는 스타일
- 너무 화려하거나 게임화된 스타일
- 너무 고가/프리미엄 교육 서비스처럼 무거운 스타일

## 16. 리디자인 이후 기대 상태

리디자인 이후 사용자는 아래처럼 느껴야 한다.

- “생각보다 쉽게 시작할 수 있다”
- “짧게 말해도 충분히 연습이 된다”
- “돈을 내고 써도 아깝지 않을 것 같다”
- “전체적으로 정돈돼 있고 믿을 만하다”

즉 이번 리디자인의 성공은 예쁘게 보이는 것보다 `더 쉽게 믿고, 더 쉽게 시작하고, 더 쉽게 결제하게 만드는 것`으로 판단한다.
