## 1. Clerk 설정 (최우선 🔴)

> Clerk는 로그인/회원가입을 담당하는 서비스.
> 이게 없으면 앱이 흰 화면만 표시됨.

---

### 1-1. Clerk 계정 만들기

1. 브라우저에서 **https://clerk.com** 접속
2. 우측 상단 **Sign up** 클릭
3. Google 계정 또는 이메일로 회원가입
4. 가입 완료 후 대시보드로 자동 이동됨

---

### 1-2. 앱(Application) 만들기

1. 대시보드 상단에 **"Create application"** 버튼 클릭
2. **Application name** 입력란에 `LinguaCall` 입력
3. 아래 로그인 방식 선택 화면에서:
   - **Email** → 체크
   - **Google** → 체크
   - 나머지는 지금 당장 필요 없음
4. **Create application** 버튼 클릭
5. 앱이 생성되고 빠른 시작 가이드 화면이 뜸 → 무시하고 진행

---

### 1-3. API 키 복사하기

> 이 키들이 코드에서 Clerk를 연결하는 핵심값.

1. 좌측 사이드바에서 **API Keys** 클릭
2. 아래 두 값을 메모장에 복사해서 보관:

   **Publishable key** (공개키, 프론트엔드용)
   ```
   pk_test_xxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
   → Vercel 환경변수에서 사용

   **Secret key** (비밀키, 백엔드용)
   - "Secret keys" 섹션 우측 **눈 아이콘** 클릭해서 값 표시 후 복사
   ```
   sk_test_xxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
   → Railway 환경변수에서 사용

   > ⚠️ Secret key는 절대 외부에 노출하면 안 됨. Git에 커밋 금지.

---

### 1-4. 프로덕션 도메인 추가

> 배포된 앱의 URL에서 Clerk 로그인이 작동하도록 허용하는 설정.

1. 좌측 사이드바 → **Domains** 클릭
2. **Add domain** 버튼 클릭
3. Vercel에 배포된 앱 URL 입력 (예: `linguacall.vercel.app`)
   - Vercel 배포가 아직 안 됐으면 일단 건너뛰고 나중에 추가
4. **Continue** → 도메인 상태가 **Active** 로 표시되면 완료

---

### 1-5. Google 로그인 활성화 (개발 테스트용)

1. 좌측 사이드바 → **User & Authentication → Social connections** 클릭
2. 목록에서 **Google** 찾기 → 우측 토글 클릭해서 **Enable**
3. 팝업이 뜨면 **Enable** 버튼 클릭
4. 별도 Google 설정 없이 **Clerk 개발용 공유 키**로 즉시 동작함
   - 실 서비스 오픈 전에는 Google Cloud Console에서 직접 OAuth 앱 만들어야 함 (지금은 불필요)

---

### 1-6. Kakao 로그인 활성화 (Custom OAuth)

> **순서 중요**: Clerk 먼저 열어서 Redirect URI 확인 → 카카오 등록 → 다시 Clerk에 입력

---

#### Step 1 — Clerk Redirect URI 먼저 확인하기

> Custom Provider 입력 화면에 Redirect URI가 안 보일 수 있음.
> 아래 방법으로 직접 확인 후 메모해두기.

**방법 1 — API Keys 페이지에서 확인 (가장 빠름)**

1. Clerk 대시보드 좌측 메뉴 → **API Keys** 클릭
2. **Frontend API** 항목의 값 확인
   ```
   예시: https://known-elk-13.clerk.accounts.dev
   ```
3. 이 URL 뒤에 `/v1/oauth_callback` 을 붙이면 Redirect URI
   ```
   예시: https://known-elk-13.clerk.accounts.dev/v1/oauth_callback
   ```
4. 이 값을 메모해두기 → 카카오 Redirect URI 등록에 사용

---

**방법 2 — Custom Provider 일단 저장 후 확인**

1. Clerk 대시보드 → **Social connections → Add custom provider**
2. **Name** 에 `Kakao` 만 입력하고 나머지 빈칸인 채로 **Save** 클릭
3. 저장된 Kakao 항목 클릭 → 상세 페이지에 **Redirect URI** 표시됨
4. 값 복사 후 나중에 나머지 항목 채워서 다시 저장

---

#### Step 1-b — Clerk에 Custom Provider 입력

Redirect URI 확인이 끝났으면 아래 입력:

1. Clerk 대시보드 → **Social connections → Add custom provider** (또는 이미 저장된 Kakao 클릭)
2. 아래 표대로 입력:

   | 항목 | 입력값 |
   |---|---|
   | **Name** | `Kakao` |
   | **Authorization URL** | `https://kauth.kakao.com/oauth/authorize` |
   | **Token URL** | `https://kauth.kakao.com/oauth/token` |
   | **Userinfo URL** | `https://kapi.kakao.com/v2/user/me` |
   | **Scopes** | `profile_nickname profile_image account_email` |
   | **Client ID** | _(Step 2에서 카카오에서 발급 후 입력)_ |
   | **Client Secret** | _(Step 4에서 카카오에서 발급 후 입력)_ |

3. **Save** 클릭

---

#### Step 2 — 카카오 디벨로퍼스 앱 등록

1. **https://developers.kakao.com** 접속 → 카카오 계정으로 로그인
2. 상단 메뉴 **내 애플리케이션 → 애플리케이션 추가하기** 클릭
3. 아래 입력 후 **저장**:
   - 앱 이름: `LinguaCall`
   - 사업자명: 본인 이름 또는 회사명
4. 생성된 앱 클릭 → 상단 **앱 키** 탭 클릭
5. **REST API 키** 복사 (Client ID로 사용)
   ```
   예시: 1a2b3c4d5e6f7a8b9c0d
   ```

---

#### Step 3 — 카카오 로그인 활성화 및 Redirect URI 등록

1. 좌측 메뉴에서 **카카오 로그인** 클릭
2. 페이지 상단 **활성화 설정** 항목 → 스위치 **ON** 클릭
   - ⚠️ **ON 하기 전까지는 Redirect URI 입력 칸이 아예 안 나타남**
   - ON 하면 페이지가 새로고침되면서 아래에 추가 항목들이 나타남
3. 스크롤 내리면 **Redirect URI** 섹션 나옴
4. **추가** 버튼 클릭 → 입력창에 Clerk Redirect URI 붙여넣기
   ```
   https://known-elk-13.clerk.accounts.dev/v1/oauth_callback
   ```
   (본인 Clerk Frontend API 기준으로 교체)
5. 입력창 우측 **저장** 버튼 클릭
   - 목록에 방금 입력한 URI가 추가되면 완료

---

#### Step 4 — Client Secret 발급

1. 좌측 메뉴 **보안** 탭 클릭
2. **Client Secret** 섹션 → **코드 생성** 버튼 클릭
3. 생성된 값 복사해두기
   ```
   예시: AbCdEfGhIj1234567890
   ```
4. **상태** 를 **사용함** 으로 변경 후 **저장**

---

#### Step 5 — 동의항목 설정

1. 좌측 메뉴 **카카오 로그인 → 동의항목** 탭 클릭
2. 아래 항목 설정:
   - **닉네임** → **필수 동의** 클릭
   - **프로필 사진** → **선택 동의** 클릭
   - **카카오계정(이메일)** → **필수 동의** 클릭
     - 비활성화 상태라면: 좌측 메뉴 **비즈니스 → 비즈 앱 전환** 먼저 진행
     - 비즈 앱 전환은 무료, 사업자 등록 없이 개인도 가능
3. **저장** 클릭

---

#### Step 6 — Clerk로 돌아가서 키 입력 후 저장

1. Clerk 대시보드 → **Social connections** → 아까 입력하던 Kakao Custom provider 화면
2. **Client ID** 칸에 카카오 **REST API 키** 붙여넣기
3. **Client Secret** 칸에 카카오 **Client Secret** 붙여넣기
4. **Save** 클릭
5. Kakao 항목이 목록에 **Enabled** 상태로 표시되면 완료

## 2. Supabase 마이그레이션 실행 (🔴 필수)

> OTP 인증이 이 테이블 없이는 동작하지 않음.

Supabase 대시보드 → **SQL Editor** → 아래 SQL 전체 복사 후 **Run** 클릭:

```sql
CREATE TABLE IF NOT EXISTS phone_verifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  clerk_user_id TEXT NOT NULL,
  phone TEXT NOT NULL,
  code TEXT NOT NULL,
  attempts INTEGER NOT NULL DEFAULT 0,
  expires_at TIMESTAMPTZ NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_phone_verifications_clerk_user_id
  ON phone_verifications (clerk_user_id);
```

실행 후 **Table Editor**에서 `phone_verifications` 테이블이 생성됐는지 확인.

## 3. Sentry 설정

> Sentry는 앱에서 에러가 발생했을 때 실시간으로 알려주는 모니터링 서비스.

---

### 3-1. Sentry 계정 만들기

1. **https://sentry.io** 접속
2. **Get Started** 클릭
3. GitHub 또는 이메일로 회원가입
4. 가입 후 **Organization name** 입력: `linguacall`
5. **Create Organization** 클릭

---

### 3-2. API용 프로젝트 만들기

1. 좌측 사이드바 → **Projects** 클릭
2. 우측 상단 **Create Project** 클릭
3. 플랫폼 목록에서 **Node.js** 선택 (검색창에 `node` 입력하면 빠름)
4. 하단 설정:
   - **Set your default alert settings**: `Alert me on every new issue` 체크 권장
   - **Project name**: `linguacall-api`
   - **Team**: 기본값 그대로
5. **Create Project** 클릭
6. 이후 뜨는 설치 가이드 화면에서 **DSN** 값 찾기:
   - 페이지에 `dsn: "https://xxx@oXXX.ingest.sentry.io/XXX"` 형태로 표시됨
   - 이 URL 전체를 복사해서 보관
   - 못 찾겠으면: 프로젝트 **Settings → Client Keys (DSN)** 탭에 있음

---

### 3-3. Web용 프로젝트 만들기

1. 다시 **Projects → Create Project** 클릭
2. 플랫폼 목록에서 **React** 선택
3. **Project name**: `linguacall-web`
4. **Create Project** 클릭
5. DSN 복사 (API 프로젝트 DSN과 **다른 값**임)

---

### 3-4. 이메일 알림 설정 (권장)

1. 좌측 사이드바 상단 본인 아이콘 → **User Settings** 클릭
2. **Notifications** 탭
3. **Issue Alerts** → **Always** 선택
4. **Save Changes**
   - 이제 에러 발생 시 가입 이메일로 알림이 옴

---

## 4. Toss Payments 설정

> Toss는 한국 결제를 담당하는 서비스.
> 테스트 계정은 사업자 등록 없이 즉시 만들 수 있음.

---

### 4-1. Toss 개발자 계정 만들기

1. **https://developers.tosspayments.com** 접속
2. 우측 상단 **시작하기** 클릭
3. 이메일 또는 카카오로 회원가입
4. 가입 완료 후 자동으로 대시보드로 이동

---

### 4-2. 테스트 키 복사하기

1. 대시보드 좌측 메뉴 → **개발 연동 → 내 개발정보** 클릭
2. 상단 탭에서 **테스트** 선택 (기본으로 선택되어 있을 수 있음)
3. 아래 두 값 복사해서 보관:
   - **클라이언트 키**: `test_ck_xxxxxxxxxx` → Railway `TOSS_CLIENT_KEY`
   - **시크릿 키**: `test_sk_xxxxxxxxxx` → Railway `TOSS_SECRET_KEY`

---

### 4-3. Webhook 등록

> 결제가 완료됐을 때 우리 서버에 자동으로 알려주는 설정.

1. 좌측 메뉴 → **개발 연동 → 웹훅** 클릭
2. **웹훅 추가** 버튼 클릭
3. 아래 입력:
   - **웹훅 URL**:
     ```
     https://your-api.up.railway.app/billing/webhooks/toss
     ```
     (`your-api.up.railway.app` 부분을 실제 Railway API 서비스 URL로 교체)
   - **이벤트**: `PAYMENT_STATUS_CHANGED` 체크
4. **저장** 클릭
5. 저장 후 **테스트 전송** 버튼 클릭
   - Railway 대시보드 → API 서비스 → **Logs** 탭에서
     `POST /billing/webhooks/toss` 로그가 찍히면 정상 연결된 것

---

### 4-4. 프로덕션 전환 (실 결제 오픈할 때)

> 지금 당장 필요 없음. 테스트 환경 확인 후 진행.

필요 서류:
- 사업자등록증 사본
- 통장 사본 (대표자 또는 법인 명의)
- 서비스 URL (실제로 접근 가능한 결제 화면 있어야 함)

신청 경로: 대시보드 → **비즈니스 전환 신청**
심사 완료 후 **라이브** 탭에서 프로덕션 키 발급 → Railway 환경변수에서 기존 테스트 키를 교체.

## 5. Railway 환경변수 추가

> 여기서 설정하는 값들이 실제 서버에서 사용되는 설정값.
> 1~4번 단계에서 복사해둔 키들을 여기서 입력.

---

### 5-1. Railway 접속

1. **https://railway.app** 접속 → 로그인
2. 대시보드에서 LinguaCall 프로젝트 클릭
3. 서비스 목록 중 **API 서비스** (백엔드) 클릭
   - 이름이 `lingua-call-api` 또는 `api` 등으로 되어 있을 것
4. 상단 탭에서 **Variables** 클릭

---

### 5-2. 환경변수 추가 방법

- **+ New Variable** 버튼 클릭
- **NAME** 칸에 변수명, **VALUE** 칸에 값 입력
- **Add** 클릭
- 아래 목록을 하나씩 반복

---

### 5-3. 추가할 환경변수 목록

| 변수명 | 값 | 어디서 복사? |
|---|---|---|
| `CLERK_PUBLISHABLE_KEY` | `pk_test_...` | Clerk → API Keys → Publishable key |
| `CLERK_SECRET_KEY` | `sk_test_...` | Clerk → API Keys → Secret key |
| `TOSS_CLIENT_KEY` | `test_ck_...` | Toss → 내 개발정보 → 클라이언트 키 |
| `TOSS_SECRET_KEY` | `test_sk_...` | Toss → 내 개발정보 → 시크릿 키 |
| `SENTRY_DSN` | `https://xxx@oXXX.ingest.sentry.io/XXX` | Sentry → linguacall-api → DSN |
| `ALLOWED_ORIGINS` | `https://your-app.vercel.app` | Vercel 배포 URL (실제 URL로 교체) |
| `OPENAI_EVAL_MODEL` | `gpt-4o-mini` | 그대로 입력 |

---

### 5-4. 기존 변수 수정

이미 `OPENAI_EVAL_MODEL` 이 설정되어 있다면:
1. 해당 변수 클릭
2. 값을 `gpt-5.4-mini` → `gpt-4o-mini` 로 수정
3. **Save** 클릭

---

### 5-5. 재배포 확인

변수 저장 후 Railway가 자동으로 재배포 시작됨.
1. 상단 **Deployments** 탭 클릭
2. 맨 위 배포 항목 상태가 **SUCCESS** (초록색) 인지 확인
3. **FAILED** (빨간색) 이면 해당 항목 클릭 → **Build Logs** 탭에서 오류 확인

## 6. Vercel 환경변수 추가

> 프론트엔드(웹 화면)에서 사용하는 환경변수.
> `VITE_` 로 시작하는 변수만 브라우저에서 접근 가능.

---

### 6-1. Vercel 접속

1. **https://vercel.com** 접속 → 로그인
2. 대시보드에서 LinguaCall 프로젝트 클릭
3. 상단 탭에서 **Settings** 클릭
4. 좌측 메뉴에서 **Environment Variables** 클릭

---

### 6-2. 환경변수 추가 방법

1. **Key** 칸에 변수명 입력
2. **Value** 칸에 값 입력
3. **Environment** 체크박스:
   - `VITE_CLERK_PUBLISHABLE_KEY` → **Production**, **Preview**, **Development** 모두 체크
   - `VITE_SENTRY_DSN` → **Production**, **Preview** 체크 (Development는 선택사항)
4. **Save** 클릭

---

### 6-3. 추가할 환경변수 목록

| 변수명 | 값 | 어디서 복사? |
|---|---|---|
| `VITE_CLERK_PUBLISHABLE_KEY` | `pk_test_...` | Clerk → API Keys → Publishable key |
| `VITE_SENTRY_DSN` | `https://xxx@oXXX.ingest.sentry.io/XXX` | Sentry → linguacall-web → DSN |

---

### 6-4. 재배포

환경변수는 저장해도 현재 배포에는 바로 반영 안 됨. 재배포 필요:

1. 상단 탭 **Deployments** 클릭
2. 목록 맨 위 (가장 최근 배포) 항목 우측 **⋯** (점 세 개) 클릭
3. **Redeploy** 클릭
4. 팝업에서 **Use existing Build Cache** 체크 **해제**
5. **Redeploy** 클릭
6. 배포 상태가 **Ready** 로 바뀌면 완료

---

### 6-5. 적용 확인

배포 완료 후 Vercel이 제공하는 URL로 접속.
- Clerk 로그인 화면이 뜨면 ✅ 정상
- 흰 화면이 뜨면 → `VITE_CLERK_PUBLISHABLE_KEY` 값 다시 확인
