# 브라우저 테스트 체크리스트

## 목적

프론트 리디자인 이후 실제 브라우저에서 확인해야 할 핵심 사용자 흐름을 정리한다.
핵심 목표는 `결제 전환율 상승`에 직접 영향을 주는 신뢰감, CTA 가시성, 오류 복구, 실제 사용 흐름을 검증하는 것이다.

## 테스트 환경

- 데스크톱 브라우저 1종
  - Chrome 권장
- 모바일 브라우저 1종
  - iPhone Safari 또는 Android Chrome 중 최소 1개
- 테스트용 계정
  - OTP 로그인 가능한 전화번호
- 테스트용 결제
  - Toss sandbox 연결 상태

## 1. 첫 화면 인상

경로:

- `/`

체크:

- 로고와 서비스 설명이 허술해 보이지 않는가
- `짧고 가볍게 꾸준히 연습` 메시지가 첫 화면에서 이해되는가
- 메인 CTA가 바로 보이는가
- 모바일에서 문구가 지나치게 길거나 빽빽하지 않은가
- 하단 정책 링크가 화면을 가리지 않는가

통과 기준:

- 10초 안에 서비스 목적과 다음 행동을 이해할 수 있다

## 2. OTP 로그인

경로:

- `/verify`

체크:

- 전화번호 입력 필드가 자연스러운가
- 인증번호 요청 버튼이 화면 안에 명확히 보이는가
- 요청 후 상태 메시지가 읽기 쉬운가
- OTP 입력 단계 전환이 명확한가
- 잘못된 번호나 코드 입력 시 오류 문구가 이해 가능한가

통과 기준:

- 처음 보는 사용자도 흐름을 막힘 없이 따라갈 수 있다

## 3. 로그인 유지

체크:

- OTP 인증 후 `/session`으로 바로 이동하는가
- 브라우저 새로고침 후 다시 OTP를 요구하지 않는가
- 다시 접속해도 세션이 유지되는가
- 로그아웃 후 보호 페이지에 들어가면 다시 로그인 화면으로 이동하는가

통과 기준:

- 재방문 UX가 끊기지 않는다

## 4. 세션 홈

경로:

- `/session`

체크:

- `지금 시작` CTA가 가장 먼저 보이는가
- 예약, 최근 리포트, 현재 플랜 정보가 과밀하지 않은가
- 세션 생성 카드가 첫 화면을 과도하게 차지하지 않는가
- 모바일에서도 카드 간격과 버튼 크기가 자연스러운가

통과 기준:

- 사용자가 망설이지 않고 다음 행동을 고를 수 있다

## 5. 세션 생성

체크:

- 언어, 레벨, 주제, 길이 선택이 자연스러운가
- 일본어, 프랑스어 포함 각 언어 선택이 정상 동작하는가
- 즉시 세션 생성 후 목록과 상태가 자연스럽게 갱신되는가
- 예약 생성이 즉시 시작과 시각적으로 구분되는가

통과 기준:

- 세션 생성 자체가 복잡해 보이지 않는다

## 6. 실시간 통화

체크:

- 마이크 권한 요청이 자연스러운가
- 연결 중 상태 문구가 이해 가능한가
- transcript가 무리 없이 노출되는가
- 종료 버튼이 명확한가
- 종료 후 즉시 상태가 반영되는가

통과 기준:

- 통화 시작과 종료가 불안정해 보이지 않는다

## 7. 결제 화면

경로:

- `/billing`

체크:

- 현재 플랜과 업그레이드 선택지가 한눈에 구분되는가
- 각 플랜 차이가 읽기 쉬운가
- 가격, 포함 분수, 세션 길이가 빠르게 이해되는가
- 추천 플랜 강조가 과하지 않은가
- 무료 사용자도 압박감보다 자연스러운 업그레이드 제안으로 느끼는가

통과 기준:

- 단순 가격표가 아니라 설득력 있는 제안처럼 보인다

## 8. Toss 결제

체크:

- 결제 버튼 클릭 시 Toss sandbox 흐름이 시작되는가
- 성공 후 `/billing` 복귀와 성공 배너가 자연스러운가
- 현재 플랜 상태가 즉시 갱신되는가
- 실패나 취소 후 복구 문구가 읽기 쉬운가

통과 기준:

- 결제 전환 흐름이 끊기지 않는다

## 9. 리포트 화면

경로:

- `/report/:reportId`

체크:

- 총점, 요약, 추천 액션이 상단에서 바로 보이는가
- 상세 분석은 읽고 싶을 때 자연스럽게 내려가며 볼 수 있는가
- 카드, 배지, 여백이 세션/결제 화면과 같은 톤으로 보이는가
- 모바일에서도 점수와 카드가 깨지지 않는가

통과 기준:

- 리포트가 단순 점수표가 아니라 학습 피드백으로 느껴진다

## 10. 정책 페이지

경로:

- `/privacy`
- `/terms`

체크:

- 인코딩 깨짐이 없는가
- 문서 레이아웃이 단정한가
- 제목, 섹션, 본문이 읽기 쉬운가

통과 기준:

- 정책 페이지가 서비스 신뢰를 깎지 않는다

## 11. 반응형 최종 점검

데스크톱:

- 카드 폭이 지나치게 넓거나 비어 보이지 않는가
- 좌우 여백이 자연스러운가

모바일:

- CTA가 첫 화면 안에 보이는가
- 버튼이 너무 작지 않은가
- 가로 스크롤이 발생하지 않는가
- 입력창과 배너가 서로 겹치지 않는가

## 완료 기준

아래를 모두 만족하면 브라우저 QA 통과로 본다.

- 로그인, 세션, 결제, 리포트, 정책 페이지가 모두 정상 표시
- OTP 재로그인 강제가 사라짐
- 세션 생성, 통화 시작, 통화 종료가 모두 정상
- Toss sandbox 결제가 정상
- 모바일과 데스크톱 모두 치명적 레이아웃 문제 없음
